Oracle SQL Performance Analyzer

Executive Summary

Load a set of SPA HTML reports from the same task, reconcile the metric-specific results, and open an executive briefing that separates application impact from monitoring workload.

Expected Inputs
CPU Time, Elapsed Time, Buffer Gets, Optimizer Cost
Business Rule
Application SQL drives the decision, not DBSNMP noise
Workflow
Upload reports, build summary, save the generated HTML

Build Summary

You can upload duplicate files safely. The summary builder will dedupe repeated reports by task, execution pair, and metric. LLM mode requires the local Ruby server and an `OPENAI_API_KEY`.

How It Interprets SPA

  1. Extracts task metadata, execution pairs, workload totals, SQL counts, and per-SQL findings from the embedded XML model.
  2. Classifies SQL into application workload or system / monitoring workload based on schema and SQL text.
  3. Prioritizes business SQL by workload impact and execution frequency.
  4. Flags reduced confidence when the reports show execution errors, unsupported SQL, or mismatched execution pairs.
